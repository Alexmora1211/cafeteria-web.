# Épicas e Historias de Usuario (SCRUM)

## Proyecto

**CafeEDEC Express**

---

# Introducción

Este documento presenta las épicas y las historias de usuario del proyecto **CafeEDEC Express**, siguiendo la metodología ágil SCRUM. Cada historia describe una necesidad del usuario y cuenta con criterios de aceptación para validar su correcto funcionamiento.

---

# ÉPICA 1
## Consulta del menú

### Historia de Usuario HU-01

**Como** estudiante

**Quiero** consultar el menú del día

**Para** decidir qué alimentos comprar antes de llegar a la cafetería.

### Prioridad

Alta

### Criterios de aceptación

- El menú muestra el nombre del producto.
- El menú muestra el precio.
- El menú muestra una imagen.
- Solo aparecen productos disponibles.

---

### Historia de Usuario HU-02

**Como** estudiante

**Quiero** buscar productos por nombre

**Para** encontrarlos más rápidamente.

### Prioridad

Media

### Criterios de aceptación

- Existe una barra de búsqueda.
- La búsqueda muestra resultados relacionados.
- Si no existe el producto, el sistema lo indica.

---

# ÉPICA 2
## Gestión del carrito

### Historia de Usuario HU-03

**Como** estudiante

**Quiero** agregar productos al carrito

**Para** realizar mi pedido.

### Prioridad

Alta

### Criterios de aceptación

- Se pueden agregar varios productos.
- El carrito muestra cantidades.
- Se calcula el total automáticamente.

---

### Historia de Usuario HU-04

**Como** estudiante

**Quiero** eliminar productos del carrito

**Para** modificar mi pedido antes de confirmarlo.

### Prioridad

Alta

### Criterios de aceptación

- Se puede eliminar cualquier producto.
- El total se actualiza automáticamente.

---

# ÉPICA 3
## Realizar pedido

### Historia de Usuario HU-05

**Como** estudiante

**Quiero** confirmar mi pedido

**Para** enviarlo a la cafetería.

### Prioridad

Alta

### Criterios de aceptación

- El sistema genera un número de pedido.
- El pedido queda registrado.
- Se muestra un mensaje de confirmación.

---

### Historia de Usuario HU-06

**Como** estudiante

**Quiero** consultar el estado de mi pedido

**Para** saber cuándo pasar por él.

### Prioridad

Alta

### Criterios de aceptación

El sistema mostrará alguno de los siguientes estados:

- Pendiente
- En preparación
- Listo
- Entregado

---

# ÉPICA 4
## Pago

### Historia de Usuario HU-07

**Como** estudiante

**Quiero** pagar mi pedido en línea

**Para** evitar hacer fila.

### Prioridad

Alta

### Criterios de aceptación

- El sistema permite seleccionar un método de pago.
- El pago queda registrado.
- Se genera un comprobante.

---

# ÉPICA 5
## Administración

### Historia de Usuario HU-08

**Como** administrador

**Quiero** agregar nuevos productos

**Para** mantener actualizado el menú.

### Prioridad

Alta

### Criterios de aceptación

- Registrar nombre.
- Registrar precio.
- Registrar imagen.
- Registrar disponibilidad.

---

### Historia de Usuario HU-09

**Como** administrador

**Quiero** editar productos

**Para** actualizar información del menú.

### Prioridad

Alta

### Criterios de aceptación

- Modificar nombre.
- Modificar precio.
- Modificar imagen.
- Guardar cambios.

---

### Historia de Usuario HU-10

**Como** administrador

**Quiero** eliminar productos

**Para** retirar productos que ya no estarán disponibles.

### Prioridad

Media

### Criterios de aceptación

- El sistema solicita confirmación antes de eliminar.
- El producto deja de mostrarse en el menú.

Las épicas y las historias de usuario permiten organizar el desarrollo del proyecto mediante la metodología SCRUM, definiendo claramente las necesidades de los usuarios y facilitando la planificación de los sprints y las funcionalidades que tendrá CafeEDEC Express.
